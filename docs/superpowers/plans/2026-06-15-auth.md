# Auth (predictex-5gw) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Email + password login (register-and-go, no mailer), invite-code gated, so league members own their predictions and an admin can be designated.

**Architecture:** Use `phx.gen.auth` for the security plumbing (token table, `PlayerAuth` plug/on_mount, session controller, tests), then apply Phoenix's documented password-registration path, auto-confirm at registration (no email), and gate registration with a shared `LEAGUE_INVITE_CODE`. The `players` table is regenerated fresh (no data exists yet) and renumbered to precede `fixtures`/`predictions`. Leaderboard `/` stays public; magic-link/email is left dormant for a later upgrade.

**Tech Stack:** Phoenix 1.8.8, Ecto/Postgres, `phx.gen.auth`, Bcrypt (`bcrypt_elixir`), Elixir 1.20.1 / OTP 28 (via `mise exec --`).

**Spec:** `docs/superpowers/specs/2026-06-15-auth-design.md`

**Convention:** all `mix` commands run as `mise exec -- mix ...`.

```elixir
defmodule Mix.Tasks.Predictex.PromoteAdmin do
  @shortdoc "Promote a player to admin by email"
  @moduledoc "Usage: mix predictex.promote_admin <email>"
  use Mix.Task

  @requirements ["app.start"]

  alias Predictex.Repo
  alias Predictex.Accounts.Player

  @impl Mix.Task
  def run([email]), do: promote(email)
  def run(_), do: Mix.raise("Usage: mix predictex.promote_admin <email>")

  @doc false
  def promote(email) do
    case Repo.get_by(Player, email: email) do
      nil -> raise "No player with email #{email}"
      player -> player |> Ecto.Changeset.change(is_admin: true) |> Repo.update!()
    end
  end
end
```
